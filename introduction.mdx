AutoPilot is a visual workflow automation platform. You build automations by connecting nodes on a drag-and-drop canvas — no code required. Each workflow runs its nodes in the correct order, passing data between them automatically.

## How it works

Workflows are made of **nodes** placed on a visual canvas. You connect nodes together to define the order of execution. When a workflow runs, AutoPilot executes each node in topological order — the output of one node becomes available as input to the next.

There are three categories of node:

| Category | What it does |
|---|---|
| **Triggers** | Starts the workflow and provides initial data |
| **AI models** | Processes the context using a language model |
| **Notifications** | Sends data to an external service |

A workflow must have at least one trigger node. Trigger nodes include **Manual**, **HTTP Request**, **Google Form**, and **Stripe**. The **Execute Workflow** button appears on the canvas only when a Manual Trigger node is present.

Credentials (API keys, webhook URLs, bot tokens) are stored securely in your account and attached to individual nodes. They are never exposed on the canvas.
