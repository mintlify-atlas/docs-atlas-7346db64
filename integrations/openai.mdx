---
title: "OpenAI"
description: "Generate text with GPT models inside your AutoPilot workflows."
---

The OpenAI node calls the OpenAI API to generate text from a prompt and stores the response in the workflow context. You can reference that response by variable name in any downstream node. Both the system prompt and user prompt support Handlebars template syntax, so you can inject data produced by earlier steps directly into your prompts.

## Prerequisites

You need an **OpenAI** credential saved to your account before you can configure this node. The credential stores your OpenAI API key (format: `sk-...`).

<Steps>
  <Step title="Get your API key">
    Log in to [platform.openai.com](https://platform.openai.com), go to **API keys**, and create a new secret key.
  </Step>
  <Step title="Save it in AutoPilot">
    Navigate to **Dashboard → Credentials**, click **New Credential**, choose type **OpenAI**, and paste your key. See [Supported credential types](/credentials/supported-types) for details.
  </Step>
  <Step title="Select it in the node">
    Open the OpenAI node settings and pick the credential from the **API Key** picker.
  </Step>
</Steps>

## Configuration

<ParamField body="credentialId" type="string" required>
  The saved OpenAI credential whose API key is used to authenticate requests. Select from credentials of type **OpenAI**.
</ParamField>

<ParamField body="model" type="string" required default="gpt-4.1-mini">
  The OpenAI model to use for text generation. Defaults to `gpt-4.1-mini`. The dropdown lists all models available through your API key.
</ParamField>

<ParamField body="variableName" type="string" required>
  The key under which the node's output is stored in the workflow context. Must start with a letter, underscore, or `$`, and contain only alphanumeric characters, underscores, or `$`. Access the response in later nodes as `{{variableName.aiResponse}}`.
</ParamField>

<ParamField body="systemPrompt" type="string">
  Optional instructions that set the model's role and behavior. If omitted, the system prompt defaults to `"You are a helpful assistant."` Supports `{{variable}}` and `{{json variable}}` Handlebars syntax for interpolating context values.
</ParamField>

<ParamField body="userPrompt" type="string" required>
  The main input sent to the model. Supports `{{variableName.field}}` Handlebars syntax to reference output from previous nodes. Example: `Summarize the following text: {{previousStep.data}}`.
</ParamField>

<Note>
  Both `systemPrompt` and `userPrompt` are compiled through Handlebars before the API call, so any `{{variable}}` references are resolved against the current workflow context at execution time.
</Note>

## Output

On success, the node merges the following object into the workflow context under the key you specified in **Variable Name**:

```json
{
  "aiResponse": "The model's generated text response."
}
```

Reference the response in downstream nodes with `{{variableName.aiResponse}}`.

<Warning>
  If the API call fails, the node throws a non-retriable error and halts workflow execution. Check that your API key is valid, the selected model is available on your account, and your OpenAI account has sufficient quota.
</Warning>

## Use cases

<CardGroup cols={2}>
  <Card title="Summarize form responses" icon="file-lines">
    Collect a Google Form submission, pass the answers as context, and use the OpenAI node to produce a concise summary before routing it to a Slack or email node.
  </Card>
  <Card title="Classify inbound webhooks" icon="tags">
    Receive an HTTP request trigger with arbitrary payload data and use the OpenAI node to classify or label the content before branching your workflow logic.
  </Card>
  <Card title="Draft automated reports" icon="chart-bar">
    At a scheduled interval, pull data from an earlier node and prompt the model to produce a formatted markdown report, then send it via the Email node.
  </Card>
</CardGroup>
