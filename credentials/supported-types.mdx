---
title: "Supported credential types"
description: "All credential types supported by AutoPilot nodes"
---

AutoPilot supports seven credential types, each tied to a specific category of node. When you create a credential, you choose its type, and node configuration panels only show credentials of the matching type in their picker.

<Tip>
  Use descriptive names when you save credentials — for example, `OpenAI Production`, `OpenAI Testing`, or `Slack #alerts`. If you store multiple credentials of the same type (different API keys for different environments or accounts), a clear name is the only way to tell them apart in the node picker.
</Tip>

<AccordionGroup>
  <Accordion title="OpenAI">
    **Used by:** OpenAI nodes

    **Value to enter:** Your OpenAI secret API key. Keys start with `sk-` and are generated in the OpenAI platform under **API keys**.

    **Where to get it:**
    1. Go to [platform.openai.com](https://platform.openai.com) and sign in.
    2. Click your account icon in the top-right and select **API keys**.
    3. Click **Create new secret key**, give it a name, and copy the value immediately — OpenAI only shows the full key once.

    OpenAI credentials are used by the **OpenAI** node to send prompts to GPT models and receive completions. The key must belong to an account with access to the models your workflow uses (e.g., `gpt-4o`).
  </Accordion>

  <Accordion title="Anthropic">
    **Used by:** Anthropic nodes

    **Value to enter:** Your Anthropic secret API key. Keys start with `sk-ant-` and are generated in the Anthropic console under **API keys**.

    **Where to get it:**
    1. Go to [console.anthropic.com](https://console.anthropic.com) and sign in.
    2. Navigate to **Settings → API keys**.
    3. Click **Create key**, copy the value, and store it securely — it is shown only once.

    Anthropic credentials are used by the **Anthropic** node to send messages to Claude models. Ensure the API key belongs to an account with sufficient usage credits and access to the Claude models your workflow targets.
  </Accordion>

  <Accordion title="Gemini">
    **Used by:** Gemini nodes

    **Value to enter:** Your Google Gemini API key. Keys start with `AIza` and are issued through Google AI Studio.

    **Where to get it:**
    1. Go to [aistudio.google.com](https://aistudio.google.com) and sign in with your Google account.
    2. Click **Get API key** in the left sidebar.
    3. Create a new key in an existing Google Cloud project, or let AI Studio create one for you.
    4. Copy the generated key.

    Gemini credentials are used by the **Gemini** node to call the Gemini family of generative models. If your Google Cloud project has API restrictions, ensure the Gemini API is enabled and the key has permission to call it.
  </Accordion>

  <Accordion title="Discord Webhook">
    **Used by:** Discord nodes

    **Value to enter:** A Discord incoming webhook URL. URLs follow the format `https://discord.com/api/webhooks/{webhook.id}/{webhook.token}`.

    **Where to get it:**
    1. Open Discord and go to the server and channel where you want AutoPilot to post messages.
    2. Click the gear icon next to the channel name to open **Edit Channel**.
    3. Go to **Integrations → Webhooks → New Webhook**.
    4. Give the webhook a name, optionally set an avatar, then click **Copy Webhook URL**.

    Discord credentials are used by the **Discord** node to post messages to a specific channel. Each webhook URL is tied to a single channel — create separate credentials if you want to post to multiple channels.
  </Accordion>

  <Accordion title="Slack Webhook">
    **Used by:** Slack nodes

    **Value to enter:** A Slack incoming webhook URL. URLs follow the format `https://hooks.slack.com/services/T.../B.../...`.

    **Where to get it:**
    1. Go to [api.slack.com/apps](https://api.slack.com/apps) and sign in.
    2. Create a new Slack app (or open an existing one), then go to **Incoming Webhooks** and toggle it on.
    3. Click **Add New Webhook to Workspace**, choose the channel you want to post to, and click **Allow**.
    4. Copy the webhook URL from the list.

    Slack credentials are used by the **Slack** node to post messages to a workspace channel. The webhook is scoped to the channel you selected during setup — create a separate credential for each channel you want to target.
  </Accordion>

  <Accordion title="Telegram Bot">
    **Used by:** Telegram nodes

    **Value to enter:** A Telegram bot token. Tokens follow the format `{bot_id}:{alphanumeric_secret}` (for example, `123456789:ABC-DEF1234ghIkl-zyx57W2v`).

    **Where to get it:**
    1. Open Telegram and search for **@BotFather**.
    2. Send `/newbot` and follow the prompts to choose a name and username for your bot.
    3. BotFather replies with your bot token — copy it.

    Telegram credentials are used by the **Telegram** node to send messages through your bot to a chat or channel. Note that the bot must be added to the target chat and have permission to send messages before it can deliver notifications.
  </Accordion>

  <Accordion title="Email (SMTP)">
    **Used by:** Email nodes

    **Value to enter:** An app password or SMTP credential for your email account. AutoPilot currently accepts an app-specific password (not your main account password) in the value field, as shown by the placeholder `xxxx xxxx xxxx xxxx`.

    **Where to get it (Gmail example):**
    1. Go to your Google Account at [myaccount.google.com](https://myaccount.google.com).
    2. Navigate to **Security → How you sign in to Google → 2-Step Verification** and confirm it is enabled.
    3. Search for **App passwords** in your account settings.
    4. Generate a new app password for **Mail** and your device type.
    5. Copy the 16-character password Google displays.

    For other email providers (Outlook, Zoho, custom SMTP servers), refer to your provider's documentation to generate an app password or SMTP credentials. Email credentials are used by the **Email (SMTP)** node to send messages from your configured address.

    <Note>
      Using your main account password directly is not recommended and may be blocked by your email provider. Always generate a dedicated app password when available.
    </Note>
  </Accordion>
</AccordionGroup>
