---
title: "Credentials overview"
description: "Manage API keys and secrets for your integrations"
---

Credentials are saved API keys, tokens, and other secrets that AutoPilot nodes use to authenticate with external services. Instead of entering a raw key every time you configure a node, you save it once as a credential and reference it by name across any number of workflows.

## Where to find credentials

Navigate to **Dashboard → Credentials** to see all credentials saved to your account. From there you can create new ones, edit existing ones, and delete ones you no longer need. You can also search credentials by name and page through results if you have many saved.

## Adding a credential

<Steps>
  <Step title="Open the new credential form">
    From the **Credentials** dashboard, click **New Credential** in the top-right corner.
  </Step>
  <Step title="Name your credential">
    Enter a descriptive name in the **Name** field. The name is for your reference only — it appears in node dropdowns when you configure a workflow.
  </Step>
  <Step title="Choose the type">
    Select the credential type from the **Type** dropdown. The type determines which nodes can use it and what value format is expected. See [Supported credential types](/credentials/supported-types) for the full list.
  </Step>
  <Step title="Enter the value">
    Paste your API key, webhook URL, bot token, or other secret into the value field. The label and placeholder update based on the type you selected — for example, an OpenAI credential shows `sk-...` as the expected format.
  </Step>
  <Step title="Save">
    Click **Create Credential**. The credential is saved to your account and immediately available in node configuration dropdowns.
  </Step>
</Steps>

<Note>
  You can also create a credential directly from within a node's configuration panel. If a required credential doesn't exist yet, the picker provides an **Add** button that takes you to the new credential form and returns you to the workflow afterward.
</Note>

## Linking credentials to nodes

When you configure an AI node (OpenAI, Anthropic, Gemini) or a notification node (Discord, Slack, Telegram, Email), the node's settings panel includes a credential picker. Click the picker to see all credentials of the matching type saved to your account, then select the one you want. The node stores a reference to that credential — not the raw secret — so the key is never exposed on the canvas.

If the picker shows no credentials, it means none of the correct type exist yet. Click **Add** in the picker to create one without leaving the workflow editor.

## Account scoping

Credentials are private to your account. They are never shared with other users, visible to collaborators, or accessible across accounts. Each credential is tied to your user ID and can only be retrieved, updated, or deleted by you.

## Security

Credential values are stored server-side and scoped to your account. The value field is masked in the UI, and once a credential is saved the raw value is not displayed through the interface — only the credential name and type are shown.

<Warning>
  AutoPilot does not display the raw credential value after it is saved. If you lose or rotate a key, edit the credential and paste the new value to update it.
</Warning>

## Editing a credential

Click any credential in the list to open its edit form. You can change the name, type, or value. Click **Update** to save your changes. All nodes that reference this credential will immediately use the updated value on their next execution.

## Deleting a credential

To delete a credential, click the remove button on its row in the **Credentials** dashboard and confirm the action.

<Warning>
  Deleting a credential removes it from your account permanently, but does **not** delete any nodes that were using it. Those nodes will have their credential reference cleared and will fail to execute until a new credential is selected. Review which workflows use a credential before deleting it.
</Warning>
